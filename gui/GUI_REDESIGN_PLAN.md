# GUI 重设计计划（工具软件导向）

## 1. 目标与范围

### 1.1 目标
- 将当前界面改造成更典型的工具软件结构：**命令清晰、操作分区明确、状态可追踪**。
- 同时支持两类高频任务：
  - 单文件查看/微调（预览、Mip 切换、尺寸处理、保存）。
  - 批量转换（输入来源、输出策略、进度与失败可回溯）。
- 降低“同屏信息过载”，避免左侧面板承担过多职责。
- **优先做到主工作区少滚动或不滚动**，将低频项折叠收纳到菜单或抽屉中。

### 1.2 非目标（本计划先不做）
- 不改动 BLP 编解码核心逻辑（Rust/C API）。
- 不改动 shell 扩展（缩略图 DLL）业务能力。
- 不引入新的 UI 框架，继续使用 Dear ImGui + DX11。

## 2. 当前布局痛点（现状评估）

- 左侧面板同时承载：文件列表、路径配置、格式参数、转换按钮、日志，任务语义混杂。
- 右侧区域既做预览又做编辑控制，BLP 层级、缩放、尺寸操作互相挤占空间。
- 顶部工具栏偏“系统设置”，缺少“当前任务上下文”与主操作入口。
- 状态信息分散：部分在日志、部分在状态栏、部分在提示文字，不利于快速定位失败原因。
- 批量转换过程缺少更直观的任务视图（队列、成功/失败统计、可重试）。

## 3. 新信息架构（IA）

### 3.1 顶层结构
- **菜单栏（Menu Bar）**：`文件` / `编辑` / `视图` / `工具` / `帮助`
- **顶部轻量工具条（可选）**：仅保留高频按钮
- **中部工作区（Workspace）**
  - 左：资源区（文件与批量输入）
  - 中：预览舞台（图像显示与交互）
  - 右：属性检查器（当前文件操作）
- **底部任务区（Task Drawer）**
  - 转换任务进度 / 日志 / 错误列表（可折叠）
- **底部状态栏（Status Bar）**

### 3.2 设计原则
- 结构固定：用户始终知道“在哪看文件、在哪改参数、在哪看结果”。
- 任务分层：全局操作、当前文件操作、批量任务操作不混放。
- 反馈闭环：每次转换/保存都有即时反馈 + 可追溯记录。
- 默认主区“无纵向滚动条”优先，只允许列表区/日志区滚动。
- 非必要功能默认折叠或收纳进菜单，避免主界面堆按钮。

## 4. 布局重构方案

### 4.1 菜单栏 + 轻量工具条
- 菜单栏固定放置低频与系统类操作：
  - `文件`：添加文件/文件夹、清空列表、导出日志、退出
  - `编辑`：移除、全选、反选、重试失败项
  - `视图`：显示/隐藏任务区、重置布局、缩放相关
  - `工具`：BLP 关联、缩略图开关、设置
  - `帮助`：版本、更新说明、关于
- 工具条仅保留高频 4~6 个按钮（如添加、扫描、开始转换、停止）。

### 4.2 左侧资源区（建议宽 300，可拖拽）
- Tab 1：**资源列表**
  - 文件搜索、排序、筛选（格式/错误状态）。
  - 支持多选与批量操作（移除、仅转换选中项）。
- Tab 2：**输入来源**
  - 输入目录、是否递归、快速扫描。
- 左侧区内部允许滚动；主画布区域不随之滚动。

### 4.3 中央预览舞台（主区域）
- 纯预览优先：尽量减少控制项覆盖在图像上。
- 顶部轻量工具条：缩放、适应窗口、原始大小、背景显示。
- BLP 专项（Mip）作为舞台下方子区，不与核心预览争夺空间。

### 4.4 右侧属性检查器（建议宽 320）
- 分组卡片：
  - **文件信息**（尺寸、格式、文件大小、POT 提示）。
  - **图像调整**（宽高、锁定比例、预设尺寸、POT 对齐）。
  - **输出设置**（格式、质量、覆盖策略、BLP mip 规则）。
  - **保存操作**（保存当前、另存为、恢复原始）。
- 低频设置默认折叠（高级选项），避免出现长滚动面板。

### 4.5 底部任务区（默认折叠，高度 180~240）
- Tab：`任务进度` / `日志` / `错误`
- 转换期间自动展开；空闲可折叠，减少视觉噪声。
- 若用户关闭任务区，错误提示改为状态栏摘要 + 菜单项红点提醒。

## 5. 关键交互流程（重设计后）

### 5.1 单文件处理
1. 添加文件 -> 资源列表选中。
2. 中央预览显示，右侧属性可调尺寸/格式参数。
3. 执行保存，底部任务区记录结果（成功/失败原因）。

### 5.2 批量转换
1. 选择输入来源（文件夹/拖拽）。
2. 在右侧设置输出格式与策略。
3. 开始转换，任务区显示总体进度与失败项。
4. 支持按失败项重试。
5. 低频动作（导出日志、批量清理）放入菜单栏 `文件/编辑`。

### 5.3 BLP 深度操作
1. 选中 BLP 后自动展示 Mip 列表与 Alpha 控制。
2. 切换 Mip 时同步更新右侧文件信息与可保存操作。

## 6. 视觉规范（工具软件风格）

- 字体：延续微软雅黑优先，确保中文信息密度可读。
- 间距：统一 8px 网格（8/16/24）。
- 强调色：仅用于“主按钮/当前选中/关键状态”，避免全局高饱和。
- 状态色：
  - 成功：绿色
  - 警告：橙色
  - 错误：红色
  - 信息：中性蓝灰
- 控件规则：同类控件宽度统一，主次按钮层级固定。

## 7. 代码落地计划（分阶段）

### Phase 1：骨架重排（App Shell）
- 目标：先把区域框架搭好，不改业务逻辑。
- 涉及文件：
  - `gui/src/ui_main.cpp`
  - `gui/src/ui_main.h`
- 产出：菜单栏 + 轻量工具条 + 三栏工作区 + 底部任务抽屉基础布局。

### Phase 2：左侧资源区重构
- 目标：将“文件列表/输入来源”从当前左栏杂糅状态拆分为 Tab。
- 涉及文件：
  - `gui/src/ui_left_panel.cpp`（可拆分为 `ui_file_panel.cpp`）
- 产出：资源管理清晰化、筛选与多选能力预留。

### Phase 3：右侧属性检查器重构
- 目标：将预览编辑控制整理为分组卡片。
- 涉及文件：
  - `gui/src/ui_right_panel.cpp`（建议拆出 `ui_inspector_panel.cpp`）
- 产出：单文件操作路径缩短，参数更聚合。

### Phase 4：底部任务区与日志系统整合
- 目标：统一进度、日志、错误三类反馈。
- 涉及文件：
  - `gui/src/ui_left_panel.cpp`
  - `gui/src/ui_main.cpp`
- 产出：任务可追踪、错误可定位、批量转换更可控。

### Phase 5：视觉统一与交互细化
- 目标：统一样式 token、按钮层级、间距、状态色。
- 涉及文件：
  - `gui/src/style.cpp`
  - 相关 UI 文件
- 产出：整体更像成熟工具软件，而非功能拼贴界面。

## 8. 验收标准

- 布局层面
  - 用户在 3 秒内能定位“文件区/预览区/参数区/任务区”。
  - 1366x768 下关键控件无重叠、无截断。
  - 主工作区默认无纵向滚动条（文件列表与日志区除外）。
- 任务层面
  - 单文件处理链路（打开->调整->保存）不超过 3 个主要点击步骤。
  - 批量转换链路（选择输入->设置输出->开始）路径明确且可回看失败项。
- 稳定性
  - 拖拽、中文路径、BLP/TGA/PNG 打开流程不回退。
  - 现有 BLP 关联和缩略图开关能力保持可用。

## 9. 风险与对策

- 风险：状态变量仍集中在 `AppState`，UI 拆分后耦合加剧。
  - 对策：按面板建立子状态结构（资源区/预览区/任务区）。
- 风险：布局改动大，短期容易引入交互回归。
  - 对策：每个 Phase 单独可运行、可验收，不一次性大改。
- 风险：日志与任务进度来源不统一。
  - 对策：统一事件入口，先建立任务事件模型再渲染。

## 10. 执行顺序建议

1. Phase 1（先搭骨架）
2. Phase 2（资源区）
3. Phase 3（属性区）
4. Phase 4（任务区）
5. Phase 5（视觉打磨）
